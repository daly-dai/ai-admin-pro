# 步骤 2：提取模块共享上下文

> 提取 CRUD 模块的公共基础信息（数据模型、API 清单、路由关系、共享数据源），
> 作为后续每个页面分析的前提上下文。

---

## 前置条件

- [ ] `output/{module}/overview.md` 已完成且用户已确认分类（步骤 1 产出）

## 必读文件

- `templates/module-context.md`（模块共享上下文模板）
- `output/{module}/overview.md`（文件清单和关联资源路径）
- `output/project-profile.md`（了解 API 封装模式、字典获取方式等）

---

## 执行指令

### 1. 定位关联资源

从 `overview.md` 的「关联资源」表中获取：

- API 文件路径
- Store 文件路径（如有）
- 路由配置路径
- 类型定义文件路径（如有）

### 2. 提取数据模型

```
1. 优先从类型定义文件（types.d.ts / interface.ts）提取：
   - 主实体的接口定义
   - 关联实体的接口定义

2. 如无独立类型文件，从 API 文件或页面组件中推断：
   - API 函数的参数和返回类型
   - 表单数据对象的字段
   - 表格列的 dataIndex

3. 输出字段表：字段名 | 类型 | 说明 | 使用页面
```

### 3. 提取 API 清单

```
读取 API 文件，提取所有导出的 API 函数：

1. 函数名 → 对应功能（getList → 分页查询、getDetail → 详情等）
2. 请求方法 → GET / POST / PUT / DELETE
3. 接口地址 → URL
4. 参数定义 → 请求参数结构
5. 使用页面 → 从 overview.md 的文件列表推断
```

### 4. 提取路由关系

```
读取路由配置文件：

1. 各页面的路由路径
2. 路由参数（:id 等动态参数）
3. 路由 meta 信息（标题、权限、面包屑等）
4. 父子路由关系
```

### 5. 提取共享数据源

```
扫描模块内所有 Vue 文件的 <script setup>，找出被多个文件共同使用的数据源：

1. 字典数据：useDictStore / getDict 调用
   - 哪些字典 key 被多个页面使用

2. 枚举数据：静态枚举（如 statusOptions）
   - 是否在多个文件中重复定义

3. 关联数据：如品类树、部门列表等
   - 来源接口
   - 被哪些页面使用
```

### 6. 提取公共组件

```
检查 overview.md 中是否有 components/ 目录下的文件：

1. 列出模块内复用的子组件
2. 记录每个组件的用途和使用页面
```

### 7. 生成产出

按 `templates/module-context.md` 的格式，将以上信息整合输出。

---

## 产出文件

```
output/{module}/context.md
```

## 完成标准

- [ ] 数据模型：主实体字段完整列出，含类型和说明
- [ ] API 清单：所有接口地址、方法、参数已记录
- [ ] 路由关系：各页面的路由路径和参数已记录
- [ ] 共享数据源：被多个页面使用的字典/枚举/关联数据已列出
- [ ] 公共组件：模块内复用的子组件已列出（如有）
- [ ] 产出文件体积适中（建议 < 2000 字），便于每个会话加载
