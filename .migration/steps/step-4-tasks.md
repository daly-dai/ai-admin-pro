# 步骤 4：汇总生成 Task 拆解方案

> 所有页面分析完成后，汇总所有描述文档，生成可用于 React 项目开发的 Task 拆解方案。

---

## 前置条件

- [ ] `output/{module}/overview.md` 中所有文件状态均为 ✅
- [ ] 所有页面描述文档已生成

## 必读文件

- `templates/tasks.md`（Task 拆解方案模板）
- `output/{module}/overview.md`（文件清单）
- `output/{module}/context.md`（模块共享上下文）
- `output/{module}/*.md`（所有页面描述文档）

---

## 执行指令

### 1. 加载所有描述文档

依次读取 overview.md 中列出的所有产出文件，获取完整的模块描述。

### 2. 分析跨文件关联

```
在独立分析阶段，每个文件的描述是独立的。现在需要汇总关联关系：

1. API 复用关系：
   - 哪些 API 被多个页面使用（如状态字典被列表和表单共用）
   - 新增/编辑表单与列表页是否共享刷新机制

2. 组件复用关系：
   - 哪些业务组件被多个页面引用
   - 是否需要提取为独立组件

3. 路由跳转关系：
   - 列表页 → 表单页 / 详情页 的跳转方式
   - 表单提交后的返回/刷新逻辑

4. 状态共享关系：
   - 是否有跨页面的 Store 共享
   - 事件通信机制（eventBus 等）
```

### 3. 按原则拆解 Task

按以下原则将模块拆解为 Task 序列：

**拆解原则**：

- 一个 Task = 一个明确交付物
- **先数据后展示**：API/类型定义 → 列表页 → 表单页 → 详情页
- 依赖关系必须声明
- 每个 Task 应包含足够的上下文信息，使开发者不需要回看 Vue 源码

**标准 Task 类型**：

| 类型        | 典型交付物                      | 依赖   |
| ----------- | ------------------------------- | ------ |
| api         | API 模块 + 类型定义             | 无     |
| page-list   | CRUD 列表页                     | api    |
| page-form   | 新增/编辑表单（Modal/独立页面） | api    |
| page-detail | 详情展示页                      | api    |
| component   | 业务组件                        | 视情况 |
| store       | 状态管理                        | api    |

### 4. 编写 Task 详细卡片

对每个 Task 编写详细卡片，包含：

```
1. 基本信息：类型、描述、依赖
2. 来源引用：指向对应的页面描述文档（文件名 + 章节）
3. 关键信息摘要：
   - API Task：接口列表 + 类型定义
   - 列表页 Task：搜索字段数量/类型、表格列数量/特殊渲染、操作列、交互模式
   - 表单页 Task：字段数量/类型、校验规则要点、联动逻辑、提交转换
   - 详情页 Task：展示字段、子模块、操作按钮
4. 注意事项：从分析中发现的特殊逻辑或潜在风险
```

### 5. 风险项 & 待确认

```
列出迁移过程中可能遇到的问题：

1. 技术选型问题：
   - Vue 中使用的第三方库在 React 中的替代品
   - Vue 特有的封装模式在 React 中的实现方式

2. 功能差异问题：
   - 目标 React 项目的组件库（sdesign/antd）与源 Vue 项目的 UI 库在功能上的差异
   - 某些 Vue 特性在 React 中无直接对应

3. 业务确认问题：
   - 功能是否需要调整
   - 交互方式是否需要变更
```

### 6. 生成产出

按 `templates/tasks.md` 的格式输出完整的 Task 拆解方案。
