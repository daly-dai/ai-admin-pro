## 一、采集维度清单（WHAT）

### 1. 表单模式

| 维度         | 说明                                             |
| ------------ | ------------------------------------------------ |
| 模式         | 新增 / 编辑 / 新增+编辑复用                      |
| 区分方式     | 路由参数（id 是否存在）/ props（mode）/ URL 路径 |
| 承载方式     | Modal 弹窗 / Drawer 抽屉 / 独立页面              |
| 编辑回填接口 | 编辑模式下获取数据的接口地址、请求方式           |
| 编辑回填时机 | 弹窗打开时 / 页面挂载时 / watch id 变化时        |

### 2. 表单布局

| 维度     | 说明                                            |
| -------- | ----------------------------------------------- |
| 列数     | 单列 / 双列 / 三列 / 响应式                     |
| 分组     | 是否用 Card / Divider / Collapse 分区，各区标题 |
| 分步骤   | 是否有 Steps 步骤条，各步骤标题和包含的字段     |
| 标签位置 | left / top / right                              |
| 标签宽度 | 固定值或 auto                                   |

### 3. 表单字段

对每个字段采集：

| 维度        | 说明                                                                                                                                                           |
| ----------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 字段名 key  | 表单数据对象中的字段路径                                                                                                                                       |
| 标签 label  | 表单项的标签文本                                                                                                                                               |
| 组件类型    | Input / InputNumber / Select / Radio / Checkbox / Switch / DatePicker / TimePicker / Upload / RichEditor / Cascader / TreeSelect / Textarea / Rate / Slider 等 |
| 校验规则    | 必填 / 长度限制 / 正则 / 自定义校验函数的具体逻辑                                                                                                              |
| 默认值      | 脚本中表单数据的初始值                                                                                                                                         |
| placeholder | 组件的 placeholder 属性                                                                                                                                        |
| 禁用条件    | 整体禁用 / 编辑模式禁用 / 条件禁用（记录条件表达式）                                                                                                           |
| 条件显隐    | 哪些字段值会触发本字段的显示/隐藏（记录条件表达式）                                                                                                            |
| 联动逻辑    | 字段 A 变化后，字段 B 的行为变化（选项重载、值清空、变必填等）                                                                                                 |
| span 占位   | 是否占整行（如 Textarea、RichEditor）                                                                                                                          |
| 额外说明    | 提示文字（tooltip / extra）、字数限制显示等                                                                                                                    |

### 4. 下拉/选择类数据源

| 维度           | 说明                                           |
| -------------- | ---------------------------------------------- |
| 数据来源       | 字典接口 / 关联业务接口 / 静态枚举             |
| 加载时机       | 表单初始化时 / 依赖其他字段变化时 / 弹窗打开时 |
| 接口地址和参数 | URL + 参数                                     |
| 远程搜索       | 是否支持远程搜索，搜索接口和参数               |
| 数据转换       | 接口返回数据到选项格式的转换逻辑               |

### 5. 特殊组件

| 维度              | 说明                                                                    |
| ----------------- | ----------------------------------------------------------------------- |
| 文件上传          | 上传接口、文件类型限制、大小限制、最大数量、上传方式（直传/先传后提交） |
| 富文本编辑器      | 使用的编辑器库、工具栏配置、图片上传配置                                |
| 子表格/动态表单项 | 可增删行的子项目结构，每行包含的字段和校验                              |
| 自定义组件        | 项目中封装的业务组件，记录组件名称、Props 和行为                        |

### 6. 提交逻辑

| 维度         | 说明                                                                           |
| ------------ | ------------------------------------------------------------------------------ |
| 提交接口     | 新增接口地址 + 方法、编辑接口地址 + 方法                                       |
| 是否共用接口 | 新增和编辑是否共用同一接口                                                     |
| 数据转换     | 提交前对表单数据的处理（日期格式化、字段映射、嵌套结构组装、文件列表提取 URL） |
| 提交前校验   | 除标准 form.validate 外的额外校验逻辑                                          |
| 成功行为     | 关闭弹窗 / 跳转页面 / 刷新列表 / 提示消息                                      |
| 失败处理     | 错误提示方式 / 特殊错误处理逻辑                                                |
| 提交 loading | 是否有 loading 状态防止重复提交                                                |

## 二、提取指令（HOW）

### 2.1 表单模式提取

```
1. 检查组件是否通过 defineProps 接收 mode / id / data 等参数
   → 如有 id 参数 → 新增+编辑复用模式

2. 检查 <script setup> 中是否有以下判断：
   - route.params.id / route.query.id → 路由参数区分
   - props.mode === 'edit' → props 区分
   - props.data / props.record → 通过父组件传入数据

3. 检查组件的承载方式：
   - 是否被 Dialog / Modal 包裹 → 弹窗承载
   - 是否被 Drawer 包裹 → 抽屉承载
   - 是否有独立路由 → 独立页面

4. 找到编辑模式的数据回填逻辑：
   - onMounted / watch(id) 中调用详情接口 → 记录接口地址
   - 直接使用 props.data 赋值 → 父组件传入
```

### 2.2 表单字段提取

```
遍历 <template> 中表单内的每个 form-item：

1. 字段名：读取内部组件的 v-model 绑定值
   → 注意 v-model 可能是 v-model:value 或 v-model:checked 等

2. 标签：读取 form-item 的 label 属性

3. 组件类型：从组件标签判断

4. 校验规则：
   → 找到 <script setup> 中的 rules 对象
   → 匹配字段名对应的规则数组
   → 标准规则：记录 required / min / max / pattern
   → 自定义 validator：分析函数体中的校验逻辑

5. 默认值：找到表单数据对象（reactive/ref）的初始值

6. 条件显隐：检查 form-item 上的 v-if / v-show
   → 追踪条件表达式中引用的字段

7. 联动逻辑：检查内部组件的 @change
   → 追踪处理函数：是否清空其他字段值？是否重新加载数据源？
```

### 2.3 数据源提取

```
对每个 Select / Cascader / TreeSelect 组件：

1. 找到选项数据的来源变量（v-for / options 绑定）
2. 追踪到 <script setup> 中的定义：
   - const options = ref([...]) → 静态枚举
   - const options = ref([]) + API 调用赋值 → 记录接口和加载时机
   - useDictStore('xxx') → 字典数据
   - 其他 hook / composable → 记录来源和功能

3. 检查是否有依赖其他字段的加载逻辑：
   - watch(fieldA, () => { loadOptionsB() }) → 联动加载
```

### 2.4 提交逻辑提取

```
找到表单提交函数（handleSubmit / onSubmit / save 等）：

1. 找到 form.validate 调用
2. 分析校验通过后的逻辑：
   - 是否对 formData 做了转换/拷贝？
     → 日期字段格式化（dayjs().format()）
     → 字段重命名 / 删除多余字段
     → 嵌套结构组装
     → 文件列表提取 URL
   - 调用的 API 函数 → 追踪到 api 文件获取接口地址
   - 新增和编辑是否分别调用不同 API

3. 分析成功回调：
   - ElMessage.success / message.success → 提示消息
   - emit('success') / emit('close') → 通知父组件
   - router.push / router.back → 页面跳转
   - 关闭弹窗的逻辑
```
