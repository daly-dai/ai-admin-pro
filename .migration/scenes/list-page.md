## 一、采集维度清单（WHAT）

### 0. 前置条件（Page Context）— 如有

> 某些列表页需要先选择一个上下文（如表名、业务类型），才能加载搜索项和表头。

| 维度         | 说明                                               |
| ------------ | -------------------------------------------------- |
| 选择器       | 组件类型（Select / TreeSelect / Tab 等）、标签文本 |
| 数据源       | 选项来自哪个接口、参数                             |
| 默认行为     | 是否有默认选中项、是否自动加载第一个               |
| 影响范围     | 切换后影响什么：搜索项 / 表头 / 查询接口 / 全部    |
| 动态配置来源 | 字段定义来自后端接口还是前端硬编码映射             |

**动态配置型**（后端返回字段定义）：重点采集配置接口地址、返回结构、字段类型到组件的映射规则。

**静态映射型**（前端 if-else）：每种类型分别按下面的标准维度列出。

### 1. 搜索区域

| 维度        | 说明                                                                               |
| ----------- | ---------------------------------------------------------------------------------- |
| 字段名 key  | 搜索表单字段的绑定值                                                               |
| 标签 label  | 表单项的标签文本                                                                   |
| 组件类型    | Input / Select / DatePicker / RangePicker / Cascader / TreeSelect / InputNumber 等 |
| 数据源      | Select 类：选项来自字典接口 / 静态枚举 / 关联接口；是否支持搜索/多选               |
| 默认值      | 脚本中对应状态变量的初始值                                                         |
| placeholder | 组件的 placeholder 属性                                                            |
| 校验规则    | 如果有                                                                             |
| 联动逻辑    | 选择 A 字段后是否影响 B 字段的选项/显隐/值                                         |
| 条件显隐    | 展开/收起、根据权限显隐、条件判断                                                  |

### 2. 工具栏 / 操作按钮区

| 维度     | 说明                                           |
| -------- | ---------------------------------------------- |
| 按钮文本 | 按钮显示文字                                   |
| 按钮类型 | primary / default / danger / link              |
| 权限控制 | 权限指令绑定的 key                             |
| 点击行为 | 新增弹窗 / 跳转页面 / 批量操作 / 导出 / 自定义 |
| 图标     | 按钮图标（如有）                               |

### 3. 表格列

| 维度             | 说明                                                             |
| ---------------- | ---------------------------------------------------------------- |
| 字段名 dataIndex | 列的数据字段名                                                   |
| 列标题           | 列头显示文字                                                     |
| 渲染方式         | 纯文本 / Tag 标签（含颜色映射规则）/ 图片 / 链接 / 自定义 render |
| 格式化           | 金额格式化 / 日期格式化 / 百分比 / 其他                          |
| 条件渲染         | 不同数据值对应不同的渲染逻辑                                     |
| 排序             | 是否支持排序                                                     |
| 固定列           | 是否固定（left / right）                                         |
| 宽度             | width / min-width 值                                             |
| 对齐             | left / center / right                                            |

### 4. 操作列

| 维度     | 说明                               |
| -------- | ---------------------------------- |
| 按钮文本 | 编辑 / 删除 / 详情 / 启用 / 禁用等 |
| 点击行为 | 跳转页面 / 打开弹窗 / 调用接口     |
| 权限控制 | 权限指令绑定的 key                 |
| 二次确认 | 是否有确认弹窗，确认文案内容       |
| 条件显隐 | 某状态下不显示某按钮的条件         |
| 请求接口 | 删除/启用禁用等直接调接口的操作    |

### 5. 分页 & 请求

| 维度         | 说明                                              |
| ------------ | ------------------------------------------------- |
| 接口地址     | URL                                               |
| 请求方法     | GET / POST                                        |
| 参数映射     | 搜索条件到请求参数的转换逻辑（如日期拆分）        |
| 分页参数命名 | pageNum/pageSize 还是 page/size 还是 current/size |
| 默认每页条数 | 数值                                              |
| 响应结构     | { list/records/data, total } 或其他               |

### 6. 行选择 & 批量操作

| 维度         | 说明                             |
| ------------ | -------------------------------- |
| 行选择类型   | checkbox / radio / 无            |
| 批量操作按钮 | 批量删除 / 批量导出 / 批量审批等 |
| 选中限制     | 最多/最少选几条                  |

### 7. 其他交互

| 维度        | 说明                     |
| ----------- | ------------------------ |
| 行展开      | expandable 展开行内容    |
| 拖拽排序    | 是否支持行拖拽           |
| 合计行      | 是否有 summary 合计      |
| 合并行/列   | span-method 合并逻辑     |
| 斑马纹/边框 | stripe / border 样式设置 |
| 空数据展示  | 自定义空状态             |

## 二、提取指令（HOW）

### 2.1 搜索区域提取

```
逐个分析 <template> 中搜索表单内的每个 form-item：

1. 找到搜索表单区域（通常是含有「查询」「重置」按钮的 form 组件）
2. 遍历每个 form-item：
   - 字段名：读取内部组件的 v-model 绑定值
   - 标签：读取 form-item 的 label 属性
   - 组件类型：从组件标签直接判断
   - 数据源：若是 Select 类，找到 option 的 v-for 来源
     → 追踪到 <script setup> 中的数据定义
     → 是 ref 直接赋值（静态枚举）？还是 API 调用获取？
   - 默认值：在 <script setup> 中找到对应 reactive/ref 的初始值
   - 联动：检查 @change 事件处理函数，分析函数体中是否修改其他字段
   - 显隐：检查 form-item 上的 v-if / v-show 条件
```

### 2.2 工具栏提取

```
找到表格上方的按钮区域：

1. 遍历每个按钮组件：
   - 读取按钮文本（插槽内容或属性）
   - 读取 type 属性
   - 检查权限指令（v-permission / v-auth 等，参考 project-profile.md）
   - 追踪 @click 事件处理函数 → 判断行为类型
```

### 2.3 表格列提取

```
分析表格组件下的每个列定义：

1. 遍历每个 table-column：
   - 字段名：读取 prop / dataIndex 属性
   - 标题：读取 label / title 属性
   - 宽度/固定/排序：读取对应属性

2. 重点分析插槽内容（#default / v-slot）：
   - 纯文本：无插槽 → 直接显示字段值
   - Tag 标签：使用了 Tag 组件 → 记录颜色映射规则
   - 格式化：使用了 dayjs / toFixed / filter → 记录格式化方式
   - 条件渲染：插槽内有 v-if → 记录条件和对应渲染
   - 图片：使用了 img / Image 组件
   - 链接：使用了 router-link / @click 跳转
```

### 2.4 操作列提取

```
找到表格的操作列（通常是最后一列，label 为「操作」）：

1. 遍历每个操作按钮：
   - 读取按钮文本
   - 检查权限指令
   - 追踪 @click → 分析处理函数：
     - router.push → 跳转页面（记录目标路由）
     - 打开弹窗（dialog/drawer visible = true）→ 弹窗表单
     - 调用 API → 记录接口地址
     - 弹出确认框（ElMessageBox.confirm / Modal.confirm）→ 记录确认文案
   - 检查 v-if / v-show → 条件显隐逻辑
```

### 2.5 分页 & 请求提取

```
分析列表查询的核心逻辑：

1. 找到列表查询函数（getList / fetchList 等）：
   - 记录调用的 API 函数 → 追踪到 api 文件 → 记录接口地址和方法
   - 分析请求参数的组装逻辑（是否有日期拆分、字段重命名等转换）

2. 找到分页相关变量：
   - 参数命名（pageNum/pageSize 等）
   - 默认值（每页条数）

3. 找到响应处理逻辑：
   - 数据赋值（tableData = res.list / res.data.records 等）
   - 总数赋值（total = res.total）
```

### 2.6 变量追踪通用方法

```
当需要追踪某个变量的来源时：

1. v-model="formData.xxx"
   → 在 <script setup> 中找 formData 的定义
   → 找到 reactive({...}) 或 ref({...}) 中 xxx 的初始值

2. v-for="item in xxxOptions"
   → 在 <script setup> 中找 xxxOptions
   → 是 ref([...]) 静态数据？
   → 还是通过 API 获取？（找 onMounted 或 watch 中的赋值）
   → 还是通过 store/hook 获取？（如 useDictStore）

3. @change="handleXxx"
   → 在 <script setup> 中找到 handleXxx 函数
   → 分析函数体：修改了哪些变量？调用了哪些 API？触发了什么联动？

4. :disabled="someCondition"
   → 追踪 someCondition 的计算逻辑（computed / 直接表达式）
```

## 四、完整性检查清单（VERIFY）

分析完成后，逐条检查：

- [ ] 每个搜索字段都追踪到了数据源（静态枚举 / 字典接口 / API）
- [ ] 每个 Select 类字段都记录了选项来源和加载时机
- [ ] 所有表格列的插槽内容都已分析（无遗漏的 #default）
- [ ] 所有操作列按钮的 @click 处理函数都已追踪到行为
- [ ] 条件显隐逻辑（v-if/v-show）都记录了具体条件
- [ ] 权限控制指令都记录了权限 key
- [ ] 二次确认弹窗的文案已记录
- [ ] 分页参数命名和默认值已记录
- [ ] 请求参数的转换逻辑已记录（如日期 RangePicker → startDate + endDate）
- [ ] 所有 computed 属性都已列出并标注其影响
- [ ] 所有 watch 监听都已列出并标注触发的行为
- [ ] 导入的外部函数/hook 都标注了来源文件和功能
- [ ] 内嵌弹窗表单的字段和提交逻辑已描述（如有）
