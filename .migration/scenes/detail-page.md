## 一、采集维度清单（WHAT）

### 1. 数据来源

| 维度         | 说明                                                    |
| ------------ | ------------------------------------------------------- |
| 详情接口     | 接口地址、请求方式、参数（通常是 ID）                   |
| 多接口组合   | 是否有多个接口拼装数据（如基本信息 + 操作日志分开请求） |
| 数据加载时机 | onMounted / watch(id)                                   |
| 数据处理     | 接口返回后是否有数据转换逻辑                            |

### 2. 展示布局

| 维度     | 说明                                                    |
| -------- | ------------------------------------------------------- |
| 布局类型 | Descriptions 描述列表 / 自定义 Card 卡片 / 混合布局     |
| 列数     | 一列 / 两列 / 三列 / 响应式                             |
| 分区     | 多个 Card 分区 / Tab 分组 / Collapse 折叠面板，各区标题 |
| 标题区   | 页面标题、面包屑、返回按钮                              |

### 3. 展示字段

对每个展示字段采集：

| 维度      | 说明                                                                                                      |
| --------- | --------------------------------------------------------------------------------------------------------- |
| 字段名    | 数据对象中的字段路径                                                                                      |
| 标签文本  | 显示的标签                                                                                                |
| 渲染方式  | 纯文本 / Tag 标签（颜色映射）/ 图片预览 / 文件下载链接 / 金额格式化 / 日期格式化 / HTML 内容 / 自定义组件 |
| 空值处理  | 空值时显示什么（`--` / `暂无` / 隐藏整行）                                                                |
| span 占位 | 是否占整行（如备注、描述字段）                                                                            |
| 条件渲染  | 某些字段在特定条件下才显示                                                                                |

### 4. 关联子模块

| 维度            | 说明                                                  |
| --------------- | ----------------------------------------------------- |
| 子表格          | 是否包含子表格（如订单详情→商品明细），表头和数据来源 |
| 时间线/操作日志 | 是否有 Timeline 组件展示操作历史                      |
| 关联列表        | 其他关联数据的列表展示                                |
| 子模块数据来源  | 与主详情同接口 / 独立接口                             |

### 5. 页面操作

| 维度     | 说明                                             |
| -------- | ------------------------------------------------ |
| 编辑按钮 | 跳转编辑页面 / 直接切换为编辑模式 / 打开编辑弹窗 |
| 状态操作 | 审批 / 启用禁用 / 发布 / 撤回等状态变更按钮      |
| 返回按钮 | 返回列表 / 浏览器后退                            |
| 其他操作 | 打印 / 导出 / 分享等                             |
| 权限控制 | 操作按钮的权限 key                               |
| 条件显隐 | 操作按钮在特定状态下的显隐逻辑                   |

## 二、提取指令（HOW）

### 2.1 数据来源提取

```
1. 找到页面的数据加载逻辑：
   - onMounted 或 watch 中的 API 调用
   - 追踪到 api 文件 → 记录接口地址和方法

2. 检查是否有多个独立的数据请求：
   - getDetail(id) → 基本信息
   - getOperateLog(id) → 操作日志
   - getRelatedList(id) → 关联数据

3. 检查数据返回后是否有处理逻辑：
   - 字段格式化
   - 嵌套数据拆分
```

### 2.2 展示字段提取

```
分析 <template> 中的展示区域：

1. 如果使用 Descriptions 组件：
   - 遍历每个 descriptions-item
   - 字段名：#default 插槽中引用的数据路径
   - 标签：label 属性
   - 渲染方式：分析插槽内容（纯文本/Tag/格式化/图片等）
   - span：检查 span 属性

2. 如果使用自定义布局（div / Row+Col）：
   - 找到所有数据绑定点（{{ data.xxx }} / :src="data.xxx"）
   - 记录对应的标签文本（通常是相邻的 label 元素）
   - 分析渲染逻辑

3. 空值处理：
   - 检查 v-if/v-show 或 || '--' 等空值兜底逻辑
```

### 2.3 关联子模块提取

```
检查页面中是否包含：

1. 子表格（Table 组件）：
   - 记录表头列定义
   - 数据来源：来自详情接口的子字段（data.items）还是独立接口
   - 分析列的渲染逻辑

2. 时间线（Timeline 组件）：
   - 数据来源
   - 每条记录的展示格式

3. 其他关联展示组件
```

### 2.4 页面操作提取

```
找到页面中的操作按钮（通常在顶部或底部）：

1. 遍历每个按钮：
   - 读取文本
   - 追踪 @click → 分析行为
   - 检查权限指令
   - 检查 v-if / v-show → 条件显隐
```
