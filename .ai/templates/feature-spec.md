# Prompt: 功能规格书生成

> **用途**：spec-gen 模式的输出模板，AI 根据 Swagger + PRD 合并生成
> **流程位置**：对应 AGENTS.md 阶段②：接口合并（分支A）
> **参考文件**：`.ai/conventions/feature-spec-workflow.md`（合并工作流规范）

---

## 使用方式

1. AI 接收到 Swagger 文档和 PRD（产品文档）后，按 `feature-spec-workflow.md` 解析双源
2. 按本模板结构填充合并结果，每个信息项标注来源
3. 差异报告（第五章）中的「用户决策」列留空，等待用户填写
4. 输出文件路径：`specs/[feature]/feature-spec.md`

---

## 来源标记规范

| 标记     | 含义     | 说明                      |
| -------- | -------- | ------------------------- |
| `[S]`    | Swagger  | 信息来自 Swagger/接口文档 |
| `[P]`    | PRD      | 信息来自产品需求文档      |
| `[M]`    | 合并推断 | AI 根据双方信息推断得出   |
| `[S][P]` | 双方一致 | 两个源中均有且一致        |

## 差异状态定义

| 状态      | 含义                       | 处理方式                  |
| --------- | -------------------------- | ------------------------- |
| ✅ 一致   | 双方定义吻合               | 无需处理                  |
| ⚠️ 需确认 | 仅单方存在或有差异但不阻塞 | AI 给建议，用户确认或覆盖 |
| 🔴 冲突   | 双方定义矛盾，阻塞开发     | 必须用户决策后才能继续    |
