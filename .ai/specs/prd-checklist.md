# PRD 规范化提取清单（画 Demo 用）

> **用途**：AI 在画 Demo 阶段将用户提供的任意格式文档规范化为结构化信息时使用。
> 按以下 9 个章节逐项提取，缺失信息标 `[?]`，不猜测。
> 完整 PRD 模板（含填写指南）→ `.ai/specs/prd-template.md`

---

## 一、功能概述

- 功能名称（中文）
- 模块标识（英文，用于目录/路由/API 路径）
- 一句话描述
- 所属业务域

## 二、目标与边界

- 业务目标（1-3 项）
- 不做什么（明确排除项）

## 三、数据模型

- **实体字段表**：字段名 | 类型 | 必填 | 说明 | 示例值
- **枚举定义**：枚举名 | 值 | 显示文本
- **查询参数**：参数名 | 类型 | 必填 | 说明 | 对应实体字段

## 四、接口设计

- 标准 CRUD 五方法勾选（getList / getById / create / update / delete）
- 非标准接口描述（name / method / path / params / response）
- 接口偏离说明（分页参数命名、响应结构差异）

## 五、页面与交互

- **页面清单**：列表页 / 新增 / 编辑 / 详情（勾选）
- **列表页**：搜索字段（字段名+控件类型）| 表格列（dataIndex+宽度+特殊渲染）| 操作列 | 批量操作 | 表格上方按钮
- **表单**：交互模式（Modal / 页面 / Drawer）| 字段定义（字段名+标签+控件类型+必填+校验规则）| 联动规则
- **详情页**：展示字段（字段标签+dataIndex+列宽占比+特殊渲染）| 布局列数 | 分组

## 六、业务规则

- 权限与角色
- 状态流转（合法状态转换）
- 业务约束（唯一性 / 删除条件 / 层级限制）
- 默认行为（默认状态 / 默认排序 / 默认分页）

## 七、关联关系

- 依赖的模块
- 被依赖的模块
- 路由与菜单位置

## 八、设计稿

- 原型图/设计稿链接（有则必录）

## 九、补充

- 技术约束
- 后续迭代计划
- 开放问题
