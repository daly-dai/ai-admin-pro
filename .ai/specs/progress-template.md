# 进度追踪模板

> 复制到 specs/[feature]/progress.md，跟踪每个 Task 的完成状态

## 使用说明

- 完成 Level 1（代码验证）→ 标记 🟡
- 完成 Level 2（功能验证）→ 标记 🟢
- 完成 Level 3（业务验证）→ 标记 ✅
- 验证失败 → 标记 🔴 并记录问题

---

## 模板

`markdown

# [功能名称] - 开发进度

> 开始日期：YYYY-MM-DD
> 最后更新：YYYY-MM-DD

## Task 进度

| Task | 标题   | Level 1 | Level 2 | Level 3 | 备注        |
| ---- | ------ | ------- | ------- | ------- | ----------- |
| 1    | [标题] | ⬜      | ⬜      | ⬜      |             |
| 2    | [标题] | ⬜      | ⬜      | ⬜      | 依赖 Task 1 |
| 3    | [标题] | ⬜      | ⬜      | ⬜      |             |

## 状态图例

| 符号 | 含义                    |
| ---- | ----------------------- |
| ⬜   | 未开始                  |
| 🟡   | 代码验证通过（Level 1） |
| 🟢   | 功能验证通过（Level 2） |
| ✅   | 业务验证通过（Level 3） |
| 🔴   | 验证失败，需修复        |

## 问题记录

> 开发过程中遇到的问题和解决方案

### [日期] - [问题描述]

- **Task**: Task X
- **问题**: 具体描述
- **原因**: 分析
- **解决**: 处理方式
  `
