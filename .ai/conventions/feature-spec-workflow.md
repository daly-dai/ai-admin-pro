# 功能规格书合并工作流

> **阶段**: ②接口合并（分支A：完整合并流程）  
> **输入**: Swagger + PRD  
> **输出**: `specs/[feature]/feature-spec.md`  
> **模板**: `.ai/templates/feature-spec.md`

---

## 一、流程概览

```
Swagger/PRD → 解析 → 交叉对比 → 生成 feature-spec.md → 用户决策 → 生成 spec.md + progress.md
```

| 步骤            | 动作                                  | 关键产出                           |
| --------------- | ------------------------------------- | ---------------------------------- |
| 1. Swagger 解析 | 提取实体、接口、枚举                  | 内部 YAML（带 `deviations` 节）    |
| 2. PRD 规范化   | 按 `prd-template.md` 9 章结构对齐     | 规范化中间结果（标 `[P]`/`[?]`）   |
| 3. 交叉对比     | 字段匹配、接口映射、差异标记          | 差异清单（🔴冲突/⚠️需确认/🟢自动） |
| 4. 合并输出     | 生成 feature-spec.md                  | 含差异决策表，待用户填写           |
| 5. 确认衔接     | 处理决策 → 生成 spec.md + progress.md | 进入③改造适配                      |

---

## 二、核心规则

### 2.1 字段匹配（按优先级）

1. 精确匹配 `userName` ↔ `userName`
2. 大小写忽略 `userName` ↔ `username`
3. 下划线/驼峰互转 `user_name` ↔ `userName`
4. 语义匹配 `desc` ↔ `description`（置信度<80%需标注）

### 2.2 差异分级处理

| 级别      | 场景                                          | 处理                  |
| --------- | --------------------------------------------- | --------------------- |
| 🔴 冲突   | 核心功能无接口、主键冲突、状态流转矛盾        | **必须用户决策**      |
| ⚠️ 需确认 | PRD 字段 Swagger 缺失、枚举不一致、必填性冲突 | AI 给建议，用户确认   |
| 🟢 自动   | 系统字段缺失、int64→string、分页参数命名不同  | AI 直接处理并标注理由 |

### 2.3 权威性原则

| 决策项                                 | 权威来源 | 说明       |
| -------------------------------------- | -------- | ---------- |
| 字段类型、接口路径、必填性、枚举值     | Swagger  | 后端已实现 |
| 功能范围、页面交互、业务规则、显示文本 | PRD      | 产品定义   |

---

## 三、输出要求

### 3.1 feature-spec.md 结构

```markdown
## 差异汇总

- ✅ 一致: X 项
- ⚠️ 需确认: X 项
- 🔴 冲突: X 项

## 差异明细表

| 编号 | 维度 | 描述 | 严重级 | AI 建议 | 用户决策 |
| ---- | ---- | ---- | ------ | ------- | -------- |
| D-01 | A1   | ...  | 🔴     | ...     | [待填写] |

## 数据模型（合并后）

## 接口映射

## 页面设计
```

### 3.2 标注规范

- 来源: `[S]`=Swagger, `[P]`=PRD, `[M]`=合并
- 状态: `✅`一致 / `⚠️`需确认 / `🔴`冲突
- 推断项: 标 `[?]`

---

## 四、用户决策后衔接

1. **读取决策** → 按决策调整数据模型和 Task 范围
2. **生成 spec.md** → 使用 `specs/template.md` 展开 Task
3. **生成 progress.md** → 使用 `specs/progress-template.md`
4. **切换阶段** → 进入③改造适配（Read `.ai/modes/demo-refine.md`）
