# Task 拆分规约

> 收到需求 PRD 后，按此规约拆成独立可验证的 Task。
> **产物**：写入 `specs/{feature}-tasks.md`，包含进度表、Task 列表（含验收标准+状态）、执行规则、Task 复盘三问记录区（Q1-Q3）。

---

## 一、拆分顺序

### 1. 先匹配已有模板

需求中的每个功能点，优先按已有 Lane 拆分：

| 功能类型           | 匹配 Lane      | Task 类型                                 |
| ------------------ | -------------- | ----------------------------------------- |
| 列表/CRUD/增删改查 | CRUD Lane      | api → page-list → page-form → page-detail |
| 大屏/仪表盘/监控   | 大屏 Lane      | 按 dashboard-page 模板                    |
| 详情（含多Tab）    | 多Tab详情 Lane | 初始同 CRUD + 增量追加                    |

### 2. 再匹配已有配方

需求中的横切关注点，检查是否已有配方覆盖：

| 横切关注点        | 匹配配方      | 说明                               |
| ----------------- | ------------- | ---------------------------------- |
| 非标布局/左树右表 | non-crud-page | 已有配方时走配方，无配方时首次处理 |

**配方匹配的 Task 处理**：配方已沉淀的场景，不再拆独立 Task。直接在 Task 描述中标注"执行后触发 {配方名} 配方"。

### 3. 最后用非标 Lane 兜底

无法匹配任何模板或配方时 → 拆为独立 Task，类型标注为"非标"，执行时走 PRD 兜底。

---

## 二、拆分规则

### R1: 基础设施优先

`plugins/` `stores/` `router/` 等基础设施类的 Task 必须排在页面类 Task 之前。页面代码依赖基础设施就绪。

```
✅ 正确:  1.1 存储层 → 1.2 api/user → 1.3 pages/user
❌ 错误:  1.1 pages/user → 1.2 存储层
```

### R2: 一个 Task 一个输出锁

每个 Task 的输出锁文件集合必须是互斥的。禁止两个 Task 操作同一文件。

### R3: Task 粒度 = 一个 verify 能独立通过

每个 Task 必须能独立 `pnpm verify` 通过。如果 Task A 生成的代码引用了 Task B 的类型 → Task A 必须在 Task B 之后执行，且 Task B 先完成 verify。

### R4: 修改类 Task 用修改路径 T1-T8

对已有文件的增量修改（加列、加字段、改文案），不拆新 Task。走 AGENTS.md §五 修改路径。

### R5: 配方命中时标注，不拆新 Task

如果某个横切场景已有配方，不在 Task 列表中拆独立 Task。在原 Task 的"做什么"字段标注：`命中 {配方名} 配方 → 执行后自动注入`。

---

## 三、Task 文档格式

```markdown
### Task X.X — {名称}

| 项         | 内容                                             |
| ---------- | ------------------------------------------------ |
| **类型**   | api / page-list / page-form / page-detail / 非标 |
| **输出锁** | {文件路径列表}                                   |
| **模板**   | {模板路径，非标类写"无"}                         |
| **做什么** | {一句话}                                         |

**验收标准**：

- [ ] {可验证的标准}

**状态**：⬜ 待实施
```

## 四、产物文件结构

`specs/{feature}-tasks.md` 必须包含：

```markdown
## 整体进度

| 阶段 | Task 数 | 完成 | 待实施 |

## Task 列表

### Task X.X — {名称}

（每个 Task 按 §三 格式，含验收标准 + ⬜ 状态）

## 执行规则

（逐 Task 推进、输出锁纪律、配方沉淀标注）

## Task 复盘三问

（每个 Task 完成后追加 Q1-Q3 三条，禁止跳过或写"已完成"等无信息量内容）
```

## 五、配方命中的标注示例

```markdown
### Task 3.2 — pages/user 列表页

| 类型 | page-list |
| 输出锁 | src/pages/user/index.tsx |
| 做什么 | 新建用户列表页。 |
```
