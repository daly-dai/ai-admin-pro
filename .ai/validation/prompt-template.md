# AI 验证提示词模板

> 在 AI 生成代码后，使用此模板进行验证

## 使用方式

在代码生成完成后，发送以下提示词让 AI 进行自检：

---

## 验证提示词

```
请对刚才生成的代码进行完整验证，按照以下检查清单逐项确认：

## 验证目标
模块: {MODULE_NAME}
类型: {TYPE}  // api-module | list-page | form-page | detail-page

## 验证步骤

### 第一步：目标验证 (Goal Validation)
验证代码是否满足业务目标：

{根据类型选择对应的 goals 检查项}

请逐项检查并输出结果：
- [ ] 检查项 1: {描述} → {结果}
- [ ] 检查项 2: {描述} → {结果}
...

### 第二步：规范验证 (Standard Validation)
验证代码是否符合架构规范：

{根据类型选择对应的 standards 检查项}

请逐项检查并输出结果：
- [ ] 检查项 1: {描述} → {结果}
- [ ] 检查项 2: {描述} → {结果}
...

### 第三步：依赖验证 (Dependency Validation)
验证代码的依赖关系是否正确：

{根据类型选择对应的 dependencies/dataflow 检查项}

请逐项检查并输出结果：
- [ ] 检查项 1: {描述} → {结果}
- [ ] 检查项 2: {描述} → {结果}
...

### 第四步：交叉验证 (Cross Validation)
如果是页面组件，验证与 API 模块的对接：

- [ ] 导入的 API 模块路径是否正确
- [ ] 使用的 API 方法名是否正确
- [ ] 传递的参数类型是否匹配
- [ ] 处理的响应数据类型是否匹配

## 验证报告格式

请按以下格式输出验证报告：

```
## 验证报告: {模块名} - {类型}

### 摘要
- 总检查项: X
- 通过: X
- 警告: X
- 错误: X
- 状态: {通过/需修复}

### 详细结果

#### ✅ 通过的检查
1. {检查项名称} - {简要说明}

#### ⚠️ 警告项
1. {检查项名称} - {问题描述} - {建议}

#### ❌ 错误项
1. {检查项名称} - {问题描述} - {修复方案}

### 修复建议
{如果有错误，提供具体的修复代码}

### 最终代码
{修复后的完整代码}
```

请开始验证。
```

## 快速验证命令

### API 模块验证

```
请验证生成的 API 模块:
模块: {MODULE_NAME}

检查清单:
1. [目标] 所有接口是否都已实现？
2. [目标] 类型定义是否完整无 any？
3. [规范] 文件是否在 src/api/{MODULE_NAME}/ 目录？
4. [规范] API 对象是否命名为 {MODULE_NAME}Api？
5. [规范] 是否使用 request 实例发送请求？
6. [规范] 是否有 JSDoc 注释？
7. [依赖] 是否正确导入 request？

输出验证报告。
```

### 列表页面验证

```
请验证生成的列表页面:
页面: {PAGE_NAME}
对应 API: {MODULE_NAME}

检查清单:
1. [目标] 是否能正确展示数据列表？
2. [目标] 是否支持分页？
3. [目标] 是否支持搜索？
4. [规范] 是否使用 SSearchTable 或 STable + SForm.Search？
5. [规范] 是否使用 useSearchTable？
6. [数据流] 是否调用 {MODULE_NAME}Api.getList？
7. [数据流] 是否有错误处理？

输出验证报告。
```

### 表单页面验证

```
请验证生成的表单页面:
页面: {PAGE_NAME}
对应 API: {MODULE_NAME}
类型: {create|edit}

检查清单:
1. [目标] 是否能正确提交表单？
2. [目标] 是否有表单验证？
3. [目标] 编辑页是否有初始值回填？
4. [规范] 是否使用 SForm 组件？
5. [规范] 是否配置 items？
6. [数据流] 是否调用 {MODULE_NAME}Api.create/update？
7. [交叉] 表单数据类型是否与 API 一致？

输出验证报告。
```

### 详情页面验证

```
请验证生成的详情页面:
页面: {PAGE_NAME}
对应 API: {MODULE_NAME}

检查清单:
1. [目标] 是否能正确展示详情？
2. [规范] 是否使用 SDetail 组件？
3. [数据流] 是否调用 {MODULE_NAME}Api.getById？
4. [交叉] 展示字段是否与类型定义一致？

输出验证报告。
```
