# 阶段⑤：迭代修复（增量修改）

> **阶段定位**：贯穿生命周期所有阶段的最小范围修改。需求变更、bug 修复、加字段、改交互等小范围修改。
> 无论当前处于哪个阶段，只要是小修改都走本阶段流程。

**步骤**：

1. **读目标文件**：根据用户描述定位并读取要修改的文件
2. **如涉及类型变更**：同时读取 `src/api/{module}/types.ts`
3. **读错题集**：Read `.ai/pitfalls/index.md`，按「适用场景」匹配当前修改涉及的页面类型，将匹配的核心规则作为硬性约束执行。不确定时再按需 Read 对应详情文件
4. **读 sdesign 组件文档**：如修改涉及 sdesign 组件，Read 对应组件文档（`.ai/sdesign/components/{组件名}.md`），未读文档的组件禁止修改其属性
5. **最小范围修改**：只改用户提到的内容，不重构不相关的代码
6. **验证**：`pnpm verify`，最多 3 轮修复

**约束**：

- **最小修改原则**，不动用户没提到的代码
- 不需要读模板（已有代码就是最好的参考）
- 不需要 spec.md / progress.md

> 🔒 **输出锁**：本模式仅允许修改用户指定的目标文件及其直接关联的类型文件，禁止创建新文件。
