## 核心步骤

1. 读已有代码：types.ts + index.ts
2. 解析接口文档 → 对比差异 → 生成变更清单，告知用户

4. 读涉及组件的 sdesign 文档 → 按需更新页面
5. 验证：`pnpm verify`

## 约束

- 先对比再改，不是重新生成

- 新增字段是否加到页面中，需向用户确认

## 输出锁
