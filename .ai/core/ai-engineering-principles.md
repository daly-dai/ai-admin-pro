## 一、软件设计宪法

> 来源：`principles.md`（已合并）。这九条来自 Brooks/Parnas/Evans 等数十年的工程实践。

| #   | 原则                   | 判断标准                           |
| --- | ---------------------- | ---------------------------------- |
| ⑨   | **人是最终责任者**     | 画得出完整数据流吗？               |
| ①   | **复杂度不灭**         | 加这东西后，理解成本变短还是变长？ |
| ③   | **边界是宪法**         | 换一个决策只改一个模块吗？         |
| ②   | **为修改而设计**       | 删这个模块牵连几个地方？           |
| ④   | **假设一切会炸**       | 随便一个依赖挂了，系统会怎样？     |
| ⑦   | **能观测才存在**       | 凌晨出问题你能先知道吗？           |
| ⑧   | **技术债是债务**       | 还款计划是什么？                   |
| ⑤   | **先跑通再跑对再跑快** | 有端到端骨架还是还在纸上？         |
| ⑥   | **代码是给人读的**     | 陌生人十分钟能看懂吗？             |

冲突时：⑨ > ① > ③ > ② > ④ > ⑦ > ⑧ > ⑤ > ⑥

## 二、AI 执政纲领

> 基于宪法六条原则推导，回答"怎么设计 AI 管道"。随 AI 能力演进而调整。

### 1. 决策预编译 → 落地 ①③

不让 AI 在运行时做"如何工作"的决策。路由表 > 推理。步骤表 > 自由。模板 > 生成。

### 2. 错题集驱动 → 落地 ④⑧

翻车了才加规则，不预判。21 行错题集 > 全量知识库。3 个月未触发的条目降级。

### 3. 复杂度外移 → 落地 ①⑤

能用工具侧解决的复杂度不给 AI。模板 + verify + 脚本替代 AI 推理。

### 4. 约束优于自由 → 落地 ③⑨

禁止 > 建议。缩小选择空间 > 扩大自由度。输出锁 > 全项目权限。

### 5. 信息密度优先 → 落地 ⑥

每段 prompt 必须能回答"删掉这段 AI 会犯什么错"。回答不上来就删。

### 6. 能力分层适配 → 落地 ⑤

弱模型不给选择权（预编译决策 + 步骤表）。强模型不被约束拖慢（按需读文件）。

## 三、落地笔记

> 每次实战后更新。记录宪法/纲领在落地时验证了或挑战了什么。

### 2025-06 RBAC 实战

| 验证            | 说明                                                                                     |
| --------------- | ---------------------------------------------------------------------------------------- |
| ✅ 约束优于自由 | G2（禁止 axios）+ G13-G15（复杂度闸门）= 代码 0 error                                    |
| ✅ 决策预编译   | Lane 匹配表 + 配方匹配表 = 零推理负担的分发                                              |
| ✅ 错题集驱动   | Task 复盘发现 8 个卡点，归因到模板/规约/架构                                             |
| ⚠️ 信息密度优先 | `max-lines-per-function` 按文件路径区分是错误——组件/工具不是按目录分的。已改回复杂度治理 |
| ⚠️ 能力分层适配 | 弱模型路径未实战验证——当前仅设计了强模型路线                                             |
| ❌ 尚未验证     | 配方自动命中、非标 Lane 兜底、task-executor Skill                                        |
