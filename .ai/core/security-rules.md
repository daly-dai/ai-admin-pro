# 安全规则表（规则驱动，无需推理）

> 本文件定义 AI 操作的三级分类规则，所有判断基于固定规则表，不依赖模型推理。

---

## 操作分类速查表

| 操作类型                                         | 分类       | 处理方式                 |
| ------------------------------------------------ | ---------- | ------------------------ |
| 读取文件                                         | 自由操作   | 直接执行                 |
| 搜索代码                                         | 自由操作   | 直接执行                 |
| 列出目录                                         | 自由操作   | 直接执行                 |
| pnpm verify                                      | 自由操作   | 直接执行                 |
| 创建新模块文件（types.ts / index.ts / 页面文件） | 自由操作   | 直接执行                 |
| 删除已有文件                                     | 需确认操作 | 透明告知 + 等待用户确认  |
| 修改 package.json 依赖                           | 需确认操作 | 透明告知 + 等待用户确认  |
| 修改 eslint / rsbuild / tsconfig 等全局配置      | 需确认操作 | 透明告知 + 等待用户确认  |
| 修改 `.ai/` 下的规范文件                         | 需确认操作 | 透明告知 + 等待用户确认  |
| 修改不在当前输出锁范围内的文件                   | 禁止操作   | 必须拒绝，向用户说明原因 |
| 自行安装新依赖包                                 | 禁止操作   | 必须拒绝，向用户说明原因 |
| 修改 src/plugins/ 或 src/router/ 等基础设施代码  | 禁止操作   | 必须拒绝，向用户说明原因 |

## 固定权限等级（用户显式声明）

用户在对话开始时显式声明权限等级，AI 按对应规则执行：

### 保守模式

**触发**：用户说「使用保守模式」

**规则**：

- 所有写操作（修改文件、创建文件）前必须显式确认
- 删除操作必须显式确认
- 修改任何已有代码前展示 diff 并确认

### 标准模式（默认）

**触发**：用户未声明权限等级，或说「使用标准模式」

**规则**：

- 读取、搜索、列出目录：无需确认
- 创建新文件：无需确认
- 修改已有文件：需确认
- 删除文件：需确认
- 修改全局配置：需确认

### 全自主模式

**触发**：用户说「使用全自主模式」或「无需确认」

**规则**：

- 仅删除操作和修改全局配置需确认
- 其他操作直接执行
- 验证失败时自动修复（最多 3 轮）

---

## 禁止操作清单（绝对不可违反）

- 绝不在未经用户确认的情况下删除文件
- 绝不执行系统级命令（rm、format、del 等）
- 绝不读取或修改 `.env`、`.ssh`、`node_modules` 外的敏感文件
- 绝不将用户代码或数据发送到外部

---

## 范围限定规则

**判断依据**：用户请求中明确提到的交付物。未提及的不生成，不猜测，不自行扩展。

**遇到类型报错或 lint 错误时**：

- 只修复当前输出锁范围内的文件
- 不扩展修改范围
- 发现现有代码可能有问题时，告知用户而不是自行修改
- 不确定是否在输出锁范围内时，先确认再操作
