# 多Tab列表页 — 功能需求

> 非标场景（无 CRUD/大屏/多Tab详情模板覆盖），走非标 Lane 兜底。
> 后端接口暂无，`[待补充]` 标记表示需要人工后续补充。

---

## 基本描述

**一句话描述**：一个多 Tab 分类的查询列表页面，两个表格（Tab1 / Tab2）共用一个搜索栏，Tab 切换时搜索项动态增减，每个 Tab 有不同的操作按钮，各自支持行勾选进行批量操作。

**为什么需要**：同一业务域下存在两种不同类型的数据实体，它们共用一部分筛选条件（如时间范围、关键词），但各自有独特的筛选字段和操作逻辑。用 Tab 切换避免拆成两个独立页面，减少用户上下文切换成本。

---

## 用户操作流程

1. 用户进入页面，看到搜索栏 + Tab 标签 + Tab1 对应的表格
2. 用户在搜索栏输入条件（基础字段 + Tab1 专有字段），点击「查询」
3. 系统请求 Tab1 的 API，表格刷新显示数据
4. 用户点击 Tab2，搜索栏字段变化（Tab1 专有字段消失，Tab2 专有字段出现），表格切换到 Tab2 数据
5. 用户在 Tab2 表格中勾选若干行，批量操作按钮（如批量审核/导出）出现
6. 用户点击批量操作按钮，系统执行对应操作并刷新

## 交互细节

### Tab 切换

- **触发条件**：用户点击 Tab 标签
- **界面变化**：
  - 搜索栏：Tab 专有字段消失/出现（条件展开 items，非 hidden）
  - 操作按钮区：切换为该 Tab 对应的按钮组
  - 表格区域：表格数据切换为对应 Tab 的 API 数据
  - 勾选状态：切换 Tab 时清空当前勾选（跨 Tab 勾选状态隔离，避免误操作）

### 表格操作列

| Tab  | 操作按钮                                                           |
| ---- | ------------------------------------------------------------------ |
| Tab1 | 编辑（SButton actionType="edit"）、删除（Modal.confirm）、查看详情 |
| Tab2 | 编辑、启用/停用（Switch）、复制                                    |

> 每个 Tab 的 columns 独立定义，通过 `activeTab` 条件分支选择。

### 批量操作

- **触发条件**：用户在表格中勾选 ≥1 行
- **界面变化**：
  - 表格标题区（`STable tableTitle.actionNode`）出现批量操作按钮组 + "已选 N 项"文字
  - 无勾选时隐藏
- **数据处理**：
  - 每个 Tab 独立维护 `selectedRowKeys` 状态
  - 批量删除使用 `Modal.confirm` 二次确认（遵循 pitfall P005）
- **异常处理**：批量操作部分失败时，message 提示失败数量，表格刷新

### 搜索与刷新联动

- **点击"查询"**：携带当前 Tab 的完整搜索条件，请求对应 API，重置分页到第一页
- **点击"重置"**：清空所有搜索字段（base + 当前 Tab 专有），重置分页，刷新数据
- **Tab 切换**：不清空搜索条件（base 部分保持），仅字段联动 + 请求切换 Tab 的 API

### 数据为空

- 搜索无结果：STable 内置 empty 展示（默认"暂无数据"），不额外处理
- 表格加载中：STable 内置 loading，不额外处理

## 状态和规则

| 当前状态       | 用户操作     | 前置条件              | 结果状态               | 备注                    |
| -------------- | ------------ | --------------------- | ---------------------- | ----------------------- |
| Tab1 激活      | 点击 Tab2    | 无                    | Tab2 激活，Tab1 卸载   | 切换 Tab 不清空基础搜索 |
| Tab1 有勾选    | 点击 Tab2    | 有 selectedRows       | 勾选清空，Tab2 无勾选  | 跨 Tab 勾选状态隔离     |
| 搜索栏有值     | 切换 Tab     | 两个 Tab 有差异化字段 | 差异化字段消失/出现    | 用条件展开，非 hidden   |
| 表格数据展示中 | 点击批量操作 | selectedKeys.length>0 | Modal.confirm 弹窗确认 | 确认后执行批量 API      |

## 边界和约束

- **能做什么**：
  - 两个 Tab 共享搜索，搜索项按 Tab 动态显隐
  - 每个 Tab 不同的操作按钮 + 不同的 columns
  - 行勾选 + 批量操作（每个 Tab 独立勾选状态）
  - 分页、排序、搜索联动

- **不能做什么**：
  - ❌ 不使用 SSearchTable 一体化组件（降级为 useSearchTable + STable 组合）
  - ❌ 不使用 `hidden` 控制字段显隐（用条件展开 items）
  - ❌ Tab 勾选状态不跨 Tab 共享
  - ❌ 不在表格组件内直接调 API（API 在 useSearchTable 层或页面层）

- **性能约束**：无，常规管理后台数据量级（单表 < 10000 条）
