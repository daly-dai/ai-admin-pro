## 二、布局方案

### 2.1 屏幕模式

- [ ] 标准响应式（随浏览器窗口自适应）
- [ ] 全屏 16:9（1920×1080 固定比例，居中缩放）

### 2.2 区域划分（画出你的大屏分区）

```
┌──────────────────────────────────────────┐
│  Row 1: {区域名称，如"标题栏/全局筛选"}      │
├──────────┬──────────┬──────────┬─────────┤
│  Row 2:  │          │          │         │
│  {指标卡} │ {指标卡}  │ {指标卡}  │ {指标卡} │
├──────────┴──────────┼──────────┴─────────┤
│  Row 3:             │                    │
│  {趋势图，左列}      │ {分布图，右列}       │
├─────────────────────┼────────────────────┤
│  Row 4:             │                    │
│  {排行榜}            │ {明细/表格}         │
└─────────────────────┴────────────────────┘
```

每个 Row 的列宽比例（24 栅格，同屏内多行两列必须对齐）：

- Row 2（指标卡行）：`[6, 6, 6, 6]`（4 等分）
- Row 3（图表行）：`[{填写左列}, {填写右列}]`，如 `[9, 15]`
- Row 4（图表行）：`[{填写左列}, {填写右列}]`，**必须与 Row 3 相同比例**

## 六、数据源定义

### 6.1 后端接口

| 接口            | 方法 | URL（mock 可写假路径）          | 请求参数                   | 响应数据结构（关键字段）          |
| --------------- | ---- | ------------------------------- | -------------------------- | --------------------------------- |
| `getStatsByGet` | GET  | `/api/dashboard/{module}/stats` | `{ dateRange, category? }` | `{ totalSales, orderCount, ... }` |
| `getTrendByGet` | GET  | `/api/dashboard/{module}/trend` | `{ dateRange, category? }` | `[{ date, value, ... }]`          |
| `{填写}`        |      |                                 |                            |                                   |

### 6.2 数据流复杂度

| 复杂度 | 判断条件                | 你的选择 |
| ------ | ----------------------- | -------- |
| 简单   | ≤5 个接口，≤2 层嵌套    |          |
| 中等   | 5-15 个接口，2-3 层嵌套 |          |
| 复杂   | >15 个接口，>3 层嵌套   |          |

→ 这决定了数据流模式：**简单 = 页面集中获取，props 传入 / 中等 = Store 中间层 / 复杂 = Store + 分片**

## 填写示例（参考）

> 这是一个"财务概览大屏"的填写示例，只展示了前几个章节，帮你感受填写节奏。

### 一、基础信息

| 项       | 填写                                                                         |
| -------- | ---------------------------------------------------------------------------- |
| 大屏名称 | 财务概览大屏                                                                 |
| 路由路径 | `/dashboard/finance`                                                         |
| 业务描述 | 给 CFO 看的全局财务数据，包括收入、成本、利润、现金流，支持按时间/事业部筛选 |
| 场景类型 | 静态概览                                                                     |

### 二、布局方案

- [x] 标准响应式

```
┌─────────────────────────────────┐
│  Row 1: 全局筛选（时间 + 事业部）   │
├─────────┬────────┬────────┬─────┤
│ 总收入   │ 总成本   │ 净利润  │ 现金流│
├─────────┴────────┼────────┴─────┤
│  趋势图（左 9）    │ 品类占比（右 15）│
├──────────────────┼──────────────┤
│  事业部排行（左 9） │ 费用明细（右 15）│
└──────────────────┴──────────────┘
```

- Row 2：`[6, 6, 6, 6]`
- Row 3-4：`[9, 15]`

### 三、指标卡定义

| #   | 标签   | 数据字段       | 单位 | 环比 |
| --- | ------ | -------------- | ---- | ---- |
| 1   | 总收入 | `totalRevenue` | 万元 | 是   |
| 2   | 总成本 | `totalCost`    | 万元 | 是   |
| 3   | 净利润 | `netProfit`    | 万元 | 是   |
| 4   | 现金流 | `cashFlow`     | 万元 | 是   |
