# 目标 LLM 运行时配置

> **元约束文件**：仅在修改 `AGENTS.md` 或 `.ai/` 目录时参考。
>
> 内部模型日常开发工作流不需要读取本文件。

---

## 适用场景

当外部 AI 需要执行以下操作时，**必须先读取本文件**：

- 修改 `AGENTS.md` 的流程、约束或阶段定义
- 新增/修改 `.ai/modes/`、`.ai/templates/`、`.ai/conventions/` 下的文件
- 调整渐进式加载策略或文档分片方式
- 设计新的代码模板或 spec 模板

## 核心参数

| 参数           | 值                | 说明                   |
| -------------- | ----------------- | ---------------------- |
| 最大上下文窗口 | 128K tokens       | 输入 + 输出 token 总数 |
| 单次最大输出   | 4K tokens（默认） | 可配置扩展至 16K       |
| 跨会话记忆     | 不支持            | 仅支持会话内上下文记忆 |
| 对话轮次       | 无硬性限制        | 建议保持简洁以提高效率 |

## 对架构设计的约束

### 1. 上下文预算管理（128K 上限）

- **渐进式加载是必须的**：不能一次性加载所有 `.ai/` 文档，必须按阶段按需读取
- **单文件大小建议**：单个规范文件控制在 2K-5K tokens（约 1500-3500 字）
- **不要依赖「详见 xxx.md」跳转**：模型可能忽略引用，关键信息必须内联到当前文件
- **上下文分配参考**：
  - 系统指令 + 架构文档：≤ 20K tokens
  - 用户需求 + 对话历史：≤ 30K tokens
  - 代码上下文（读取的源码）：≤ 40K tokens
  - 预留给输出生成：≥ 16K tokens
  - 安全余量：≥ 22K tokens

### 2. 输出长度限制（默认 4K / 最大 16K）

- **代码模板不宜过大**：单个模板生成的代码应控制在 4K tokens 以内
- **分步输出策略**：复杂页面拆分为多步（先 types → 再 API → 再页面）
- **模板中标注断点**：模板文件可标注 `<!-- STEP BREAK -->` 提示模型分步生成

### 3. 无跨会话记忆

- **spec 文件是必要的**：功能规格书承担跨会话记忆的角色
- **progress.md 是必要的**：多步骤任务必须有进度文件，新会话可从断点恢复
- **文件即记忆**：所有决策、约定、已知问题必须沉淀到文件中

### 4. 文档编写原则

- **结构优先于叙述**：使用表格、列表、代码块，减少长段落
- **关键信息前置**：每个文件开头就是核心约束
- **避免跨文件引用链**：模型不一定会跟随「详见」引用，关键规则直接写在使用处
- **示例要精简**：代码示例只展示关键部分
