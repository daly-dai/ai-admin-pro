```mermaid
    D --> D1["开发修复/驳回问题"]
    D1 --> D2{"所有问题处理完?"}
    D2 -- 否 --> D

    E -- 驳回 --> C

    E1 --> E2{"所有技术口径审批通过\n且预览数据就绪?"}
    E2 -- 否 --> C

    E3 --> F["业务逐口径验收"]

    F1 -- 全部通过 --> F2["项目负责人确认 → 待上线"]

    F3 --> C

    F4 --> C

    G --> Finish([完成])

    B -.-> P["暂停/取消"]
    C -.-> P
    F -.-> P
```

## 二、项目状态机

```mermaid
stateDiagram-v2
    [*] --> 待澄清 : 业务人员创建项目

    待澄清 --> 开发中 : 项目负责人确认澄清完成
    开发中 --> 待验收 : 开发负责人提交验收
    待验收 --> 开发中 : 业务验收驳回
    待验收 --> 待上线 : 项目负责人确认验收通过
    待上线 --> 已上线 : 执行上线
    已上线 --> 已归档 : 系统自动

    待澄清 --> 已暂停 : 暂停
    开发中 --> 已暂停 : 暂停
    待验收 --> 已暂停 : 暂停
    已暂停 --> 待澄清 : 恢复
    已暂停 --> 开发中 : 恢复
    已暂停 --> 待验收 : 恢复

    待澄清 --> 已取消 : 取消(需审批)
    开发中 --> 已取消 : 取消(需审批)
    待验收 --> 已取消 : 取消(需审批)
    已暂停 --> 已取消 : 取消(需审批)
```

### 关键转换条件

| 转换            | 操作人                   | 前置条件                                                                                 |
| --------------- | ------------------------ | ---------------------------------------------------------------------------------------- |
| 待澄清 → 开发中 | 项目负责人               | 所有业务口径状态为「已澄清」                                                             |
| 开发中 → 待验收 | 项目开发负责人           | 所有「需技术开发」口径的技术口径均「审批通过」且「预览数据已就绪」；无「待重新评估」状态 |
| 待验收 → 待上线 | 项目负责人               | 所有口径均验收通过                                                                       |
| 待验收 → 开发中 | 业务人员                 | 至少一个口径验收不通过                                                                   |
| 待上线 → 已上线 | 项目开发负责人/管理员    | 无                                                                                       |
| 已上线 → 已归档 | 系统自动                 | 上线操作完成                                                                             |
| 任意 → 已暂停   | 项目负责人/管理员        | 项目处于待澄清/开发中/待验收                                                             |
| 任意 → 已取消   | 项目负责人(需管理员审批) | 项目处于待澄清/开发中/待验收/已暂停                                                      |

### 暂停 vs 取消对比

| 维度     | 暂停                | 取消             |
| -------- | ------------------- | ---------------- |
| 可恢复性 | ✅ 可恢复           | ❌ 不可恢复      |
| 口径锁定 | 保持锁定            | 释放锁定         |
| 版本处理 | 保持不变            | 标记为「已废弃」 |
| 流程状态 | 冻结（不可操作）    | 终止             |
| 审批要求 | 无需审批            | 需管理员审批     |
| 超时处理 | 30天提醒 / 90天预警 | 无               |

## 三、业务口径状态机

```mermaid
stateDiagram-v2
    [*] --> 草稿 : 项目创建时初始化
    草稿 --> 已提交 : 业务人员提交
    已提交 --> 已提交 : 业务修改(版本递增)
    已提交 --> 已澄清 : 项目负责人确认
    已澄清 --> 已提交 : 业务再次修改(版本递增)
    已提交 --> 已废弃 : 项目取消
    已澄清 --> 已废弃 : 项目取消
    草稿 --> 已废弃 : 项目取消
```

### 业务口径修改影响分析

当业务人员在「开发中」或「待验收」阶段修改业务口径时:

```
业务发起修改 → 检测技术口径状态
  ├── 无技术口径 → 正常修改，版本递增
  ├── 技术口径未开发/开发中 → 提示通知开发，正常修改
  ├── 技术口径审核中/待审批 → 警告修改影响，确认后修改
  └── 技术口径审批通过 → 强警告，确认后触发「待重新评估」
```

## 四、技术口径状态机

```mermaid
stateDiagram-v2
    [*] --> 未开发 : 口径关联至项目
    未开发 --> 开发中 : 开始编写
    开发中 --> 审核中 : 提交审核
    审核中 --> 待审批 : 所有问题处理完成
    待审批 --> 审批通过 : 组长审批通过
    待审批 --> 审批驳回 : 组长审批驳回
    审批驳回 --> 审核中 : 修改后重新提交
    审批通过 --> 待重新评估 : 业务口径被修改(系统自动)
    待重新评估 --> 审批通过 : 确认无需修改
    待重新评估 --> 审核中 : 修改后重新提交
    审批通过 --> 开发中 : 验收驳回(技术问题)
    未开发 --> 已废弃 : 项目取消
    开发中 --> 已废弃 : 项目取消
    审核中 --> 已废弃 : 项目取消
    待审批 --> 已废弃 : 项目取消
    审批通过 --> 已废弃 : 项目取消
    审批驳回 --> 已废弃 : 项目取消
    待重新评估 --> 已废弃 : 项目取消
```

### 组内审核 + 组长审批流程

```
提交审核 → 通知组内成员
  → 组内审核（提出问题）
    → 开发修复/驳回问题
      → 所有问题处理完? → 否 → 回到组内审核
      → 是 → 提交审批
        → 系统自动指定审批人（开发人员所在组的组长）
          → 组长查看完整信息
            ├── 通过 → 审批通过 → 生成预览数据 → 标记就绪
            └── 驳回 → 退回开发 → 修改后重新提交审核（循环）
```

### 审批委托机制

- 组长可委托同组成员代行审批
- 委托信息: 委托人、被委托人、委托时间、结束时间
- 超过48小时未审批 → 自动提醒组长及上级

## 五、口径版本控制与锁定

### 版本演进规则

```
项目A (首次创建):  V1.0 → V1.1 → V1.2 → V1.2(上线定格 ✅)
                                               ↓
项目B (正常迭代):  V2.0 → V2.1 → V2.2 → V2.2(上线定格 ✅)
                                               ↓
项目C (被取消):    V3.0 → V3.0(废弃 ❌)
                                               ↓
项目D (接续):      V4.0(版本号继续递增，跳过废弃版本)
```

### 口径锁定生命周期

```
空闲 → 项目关联 → 已锁定
  ├── 项目上线归档 → 版本生效 → 释放 → 空闲
  ├── 项目取消 → 版本废弃 → 释放 → 空闲
  ├── 项目移除关联 → 版本废弃 → 释放 → 空闲
  └── 项目暂停 → 保持锁定（冻结）
       ├── 恢复 → 继续锁定
       └── 取消 → 版本废弃 → 释放 → 空闲
```
