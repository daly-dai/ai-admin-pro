## 核心观点速查

**弱模型优先原则：**

- 先在弱模型验证，再在强模型优化
- 弱模型跑通的架构一定兼容强模型，反之不成立
- 弱模型上的挫败感是信号，不是噪音

**漏斗法则：**

- 黄金范例 > 错题速查 > 规则集 > 全量文档（信噪比从高到低）
- 弱模型只加载漏斗底部（范例 + 速查）
- 150 行是弱模型的有效注意力天花板

**范例驱动：**

- 规则是"理解后应用"（推理密集），范例是"看到后复制"（模式匹配）
- 每个页面类型维护一个黄金范例
- 非标场景做完后提取范例，持续扩大覆盖面

**错误飞轮：**

- 三层架构：自动捕获 → 模式聚合 → 人工审批
- 飞轮对 AI 完全透明，运行在基础设施层
- pitfalls/index.md 是 SSOT，Skill 自动消费
